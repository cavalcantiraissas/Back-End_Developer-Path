# Back-End Developer Path

Repositório central da minha trilha de desenvolvimento back-end. O objetivo é documentar e organizar o aprendizado de forma progressiva — dos fundamentos da programação até a construção de sistemas escaláveis e seguros em ambiente profissional.

A trilha segue uma estrutura deliberada: primeiro a base sólida em lógica e linguagem, depois a aplicação em sistemas reais com ferramentas do mercado.

---

## Estrutura da Trilha

### Fase 1 - Fundamentos
Programação estruturada e lógica computacional com Python e C.

- Algoritmos e estruturas de dados
- Manipulação de memória e ponteiros (C)
- Orientação a objetos (Python)

### Fase 2 - Back-End com Golang
Construção de sistemas back-end performáticos e tipados estaticamente.

- APIs REST com Go
- Concorrência com goroutines e channels
- Tratamento de erros e boas práticas

### Fase 3 - Banco de Dados
Modelagem e consulta de dados relacionais com SQL.

- Modelagem entidade-relacionamento
- Queries, joins, subqueries e índices
- Integração com aplicações Go

### Fase 4 - Infraestrutura e Deploy
Containerização e ambientes reprodutíveis com Docker.

- Dockerfile e docker-compose
- Variáveis de ambiente e volumes
- Deploy básico de aplicações back-end
